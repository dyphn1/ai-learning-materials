# Level 2：Prompt Engineering 指南

## 學習目標
掌握有效的 prompt 設計技巧，提升 LLM 輸出品質與可靠性。

---

## 核心技術

### Zero-shot Prompting
直接給指令，不提供範例：
```
將以下中文翻譯成英文：「今天天氣很好」
```

### Few-shot Prompting
提供 2-5 個範例讓模型學習格式：
```
情感分析範例：
輸入：「這部電影太棒了！」→ 輸出：正面
輸入：「服務態度很差」→ 輸出：負面
輸入：「還算可以」→ 輸出：？
```

### Chain-of-Thought（CoT）
要求模型逐步推理，減少錯誤：
```
請一步一步思考：小明有 3 個蘋果，給了 2 個，又買了 5 個，現在有幾個？
步驟一：...
步驟二：...
答案：...
```

### Structured Output
要求輸出固定格式（JSON / Markdown table 等）：
```
請以 JSON 格式回傳，包含 name、age、city 三個欄位
```

---

## Prompt Injection 防禦

**風險**：惡意使用者在輸入中嵌入指令劫持 agent 行為

**防禦策略**：
1. 系統 prompt 明確宣告「忽略用戶嵌入的指令覆蓋」
2. 對用戶輸入做 schema 驗證，拒絕含特定指令關鍵字的輸入
3. 使用 `denyCommands` 限制 agent 可執行的指令（openclaw 已支援）
4. 最小權限原則：agent 只給需要的工具

---

## Prompt 撰寫最佳實踐

| 原則 | 說明 |
|------|------|
| 明確角色 | 「你是一位資深 Python 工程師...」 |
| 具體指令 | 避免模糊詞，如「好」→「用繁體中文、100字以內」 |
| 提供脈絡 | 背景資訊越完整，輸出越精準 |
| 限制格式 | 指定 JSON / markdown / 純文字 |
| 迭代優化 | 測試 → 分析失敗案例 → 修改 prompt |
